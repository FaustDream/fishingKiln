# fishingKiln 艺术品页与机构型首页改版设计

## 1. 背景

当前站点已经具备研究杂志化的基础，但首页内容密度仍偏轻，艺术品分类页也仍停留在“分类专题页”的层级，缺少更强的策展表达、正式机构站风格和可展开的数据细节。

本轮改版不把站点做成营销落地页，也不退回成图片堆叠页，而是把首页和艺术品页提升为接近高校、博物馆、国际组织官网的内容首页与专题栏目页。

## 2. 目标

1. 首页从轻量导览升级为机构型内容首页。
2. 艺术品页升级为定制化策展栏目页，具备列表、卡片、数据详情和延伸阅读。
3. 新增一个跨首页和艺术品页复用的“内嵌详情”机制，不依赖弹窗也能展开结构化资料。
4. 内容必须来自联网整理后的真实公开来源，优先使用官方机构资料。
5. 页面动效应服务阅读层级与信息定位，不做装饰性大轮播。

## 3. 数据来源策略

本轮新增内容统一来自公开官方来源，并在本地整理为结构化数据：

- The Metropolitan Museum of Art Collection API
- Victoria and Albert Museum 官方文章与馆藏页
- British Museum 官方馆藏页
- UNESCO 官方页面

取数原则：

1. 优先使用可公开访问、可稳定复用的官方 API 或官方页面。
2. 每条艺术品或机构事实必须保留来源名称、来源链接与用途说明。
3. 页面文案可以做编辑性整合，但不能脱离原始资料编造事实。

## 4. 首页信息架构

首页调整为六段正式机构型结构：

1. 机构首屏：站点定位、年度脉络、两到三个主入口。
2. 当期策展导览：强调艺术品栏目、本轮重点对象与来源网络。
3. 编年主线：保留历史时间线，但让每个节点联动事实面板和相关条目。
4. 机构资料网格：展示馆藏来源、研究主题、器类分布、专题数量等高密度概览。
5. 研究与艺术品双列入口：一列走专题阅读，一列走艺术品策展阅读。
6. 来源网络与继续阅读：展示官方来源网络、路径与最近整理的对象。

首页的视觉语气应更接近“研究机构官网首页”，而不是“品牌故事封面”。

## 5. 艺术品页结构

艺术品页不再与其他四个分类页完全同构，而是升级为策展页。页面结构如下：

1. 页头：栏目名称、策展引言、来源机构摘要。
2. 重点馆藏条目：突出一组高价值对象卡片。
3. 策展列表：按主题或观看路径组织成列表型数据。
4. 观看方法：说明图像、材质、年代、传播路径怎样共同构成观看顺序。
5. 机构来源轨迹：展示对象来自哪些官方机构。
6. 延伸专题与器类回链：把研究条目和其他器类重新接回。

艺术品页的主体验应是“浏览一个正式策展栏目”，而不是“打开一个普通分类页”。

## 6. 通用内嵌详情机制

新增通用内嵌详情模块，形态不是弹窗，而是卡片下方或列表行内展开：

- 触发方式：按钮或卡片工具条
- 展开内容：年代、馆藏机构、材质、产地、观看重点、来源链接
- 适用范围：首页重点对象、艺术品页重点卡片、策展列表项

这样做的目的，是让页面保持正式连续阅读感，同时给数据更多层次，而不是频繁打断用户。

## 7. 视觉与动效方向

视觉方向：

- 更接近大学、研究院、博物馆官网的正式秩序
- 首页首屏采用深浅结合的机构型带状布局
- 使用真实器物图片和高密度内容区块建立可信度
- 保持青花相关配色，但避免单一蓝色统治整站

动效方向：

- 卡片进入视区渐显
- 时间线与详情区联动过渡
- 内嵌详情展开收起
- 数字或标签轻量高亮

禁止使用：

- 营销式大 Hero 文案
- 纯装饰轮播
- 为了显得动态而牺牲可读性的复杂动画

## 8. 数据模型调整

需要新增或扩展三类结构化数据：

1. 首页机构型模块数据
   - featuredRecords
   - institutionStats
   - networkPanels
   - currentFeatures

2. 艺术品页专属数据
   - curatorialIntro
   - collectionHighlights
   - collectionList
   - viewingAxes
   - relatedInstitutionIds

3. 通用内嵌详情数据
   - detailId
   - detailTitle
   - detailFacts
   - detailSourceName
   - detailSourceUrl

## 9. 页面级失败兜底

1. 远程来源整理不完整时，页面仍必须显示文本和来源信息，不出现空块。
2. 内嵌详情数据缺失时，不渲染展开入口。
3. 图片缺失时，回退成文字型记录卡。
4. 任何条目都必须保留来源指向，避免“来源不明”的正式页面内容。

## 10. BDD 验收场景

```gherkin
Scenario: 首页呈现机构型内容首页
  Given 用户进入首页
  When 页面完成渲染
  Then 用户能看到正式的机构首屏与策展导览
  And 用户能看到来源网络、资料统计和艺术品入口

Scenario: 用户在艺术品页浏览真实馆藏对象
  Given 用户进入艺术品分类页
  When 页面完成渲染
  Then 用户能看到重点对象卡片和策展列表
  And 每个对象都有来源机构信息

Scenario: 用户展开对象的内嵌详情
  Given 用户看到带详情入口的对象卡片
  When 用户展开详情
  Then 页面显示结构化事实与来源链接
  And 页面不需要跳出当前阅读上下文
```

## 11. 交付结论

本轮实现应把渔窑首页升级成一个具有正式机构站气质的内容入口，并把艺术品页升级成真正可持续阅读的策展栏目页。所有新增对象数据都来自联网整理后的官方公开来源，页面通过内嵌详情而不是说明书式文案来承载深度信息。
