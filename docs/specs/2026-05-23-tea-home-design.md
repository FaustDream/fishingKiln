# 茶具分类页与首页机构化升级设计

## 目标

把当前首页与 `tea.html` 从“分类入口 + 简短说明”的层级，提升为更接近高校、博物馆与国际组织官网的信息型页面。核心目标有两项：

1. 把 `tea.html` 升级为带器型谱系、文献规则、馆藏样本、资料台账和通用内嵌详情的正式茶具专题页。
2. 把 `index.html` 升级为茶具优先的机构型首页，让用户在首页就能看到茶具研究主线、馆藏来源和对象样本，而不是只看到品牌导语。

## 用户要求映射

- 要定制化：围绕“青花茶具研究站”组织内容，不做泛化模板页。
- 要有列表数据：提供茶具体系、文献规则、资料台账、研究路径等列表。
- 要有卡片数据：提供器型卡、统计卡、馆藏卡、来源卡。
- 要有动画效果：沿用现有 reveal motion，并为内嵌详情切换、卡片入场和列表激活保留轻量动画。
- 要有页面数据详情：茶具页提供专属数据详情区。
- 要有一个通用的内嵌数据详情：首页和茶具页共用同一套 detail deck 组件。
- 首页功能要丰富：保留现有时间线、研究索引与工艺模块，同时新增茶具专题焦点、馆藏样本和资料台账。
- 数据从网上获取：本次内容以 British Museum、The Met、UNESCO、Wikisource《茶经》《大观茶论》为基础整理。

## 视觉方向

- 气质：机构型、研究型、安静、正式，但保留渔窑的工艺温度。
- 结构：首屏采用“主叙事 + 数据简报 + 焦点专题”布局，中段以资料面板、馆藏卡和时间线串联，不做营销式大横幅。
- 页面层级：首页承担总入口，茶具页承担专题化阅读；两页通过共用 detail deck 保持一致阅读方式。
- 动效：只用于状态切换和滚动进入，不用装饰性大动效。

## 信息架构

### 首页

1. Hero + 机构简报
   - 品牌标题
   - 茶具专题 CTA
   - 统计卡
   - 焦点专题卡
2. 茶具专题焦点
   - 以饮法、器型、流通三条线介绍茶具专题
3. 茶具资料台账
   - 使用通用 detail deck
   - 左侧列表，右侧详情
4. 馆藏样本
   - 以实物卡片展示盏、杯、壶、杯碟等对象
5. 机构来源
   - 按博物馆、国际机构、公开古籍组织来源信号
6. 历史时间线 / 研究索引 / 研究路径 / 术语卡 / 工艺步骤
   - 保留现有站点骨架，让首页继续承担全站入口

### 茶具页

1. 专题页头
   - 茶具导言
   - 标签与页内导航
   - 摘要统计
2. 茶具体系
   - 先按饮法与器型分层，再展开具体器物卡
3. 文献规则
   - 用《茶经》《大观茶论》提炼盏、壶、注器与分饮关系
4. 通用内嵌详情
   - 复用首页 detail deck
   - 统一呈现时代、尺寸、用途、空间位置、来源
5. 馆藏样本
   - 选取 The Met / British Museum 的公开对象页样本
6. 资料台账
   - 列出古籍、馆藏、机构来源
7. 延伸研究
   - 保持与既有 `research.html` 的阅读连续性

## 内容来源与使用方式

### 博物馆 / 机构

- British Museum：明代青花茶壶、宣德龙纹碗等对象页，用来校对器型、年代和尺寸。
- The Met：建盏、高足杯、青花注器、杯碟等对象页，用来补足盏系、分饮器和外销服务线。
- UNESCO：景德镇城市系统与中国传统制茶社会实践，用来把器物放回城市与饮茶文化语境。

### 典籍

- 《茶经·四之器》：茶具集合与器物分工。
- 《大观茶论》：茶盏审美、点茶观看与器物要求。

### 处理原则

- 不复制长段原文，只提取结构化事实。
- 所有对象与资料卡保留原始链接。
- 文献内容用现代中文归纳，保留少量必要短句，不堆砌古文。

## 组件边界

- `renderFeatureSpotlightSection()`：首页焦点专题区，替换当前花器专场语义。
- `renderDetailDeck()`：继续作为首页与茶具页共用的详情组件。
- `renderTypologyCards()`：渲染茶具体系卡与花器等同类专题卡。
- `renderCollectionHighlights()`：渲染馆藏样本卡。
- `renderResearchLedger()`：渲染来源台账。
- `initDetailDecks()`：统一处理索引与详情面板切换。

## 验收标准

1. 首页首屏能直接看出站点当前重点是茶具专题，而不是单纯品牌页。
2. 茶具页同时具备：
   - 列表型数据
   - 卡片型数据
   - 通用内嵌详情
   - 官方来源链接
3. 通用 detail deck 至少在首页和茶具页各使用一次。
4. 自动化测试覆盖首页新结构、茶具页新结构和共享 detail deck。
5. README、计划文档和 `docs/changelog-ai.md` 同步记录本次数据结构和验证方式变化。
